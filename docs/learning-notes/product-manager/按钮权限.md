问题：APP有按钮权限吗


有，**APP 也可以有按钮权限**，只是很多团队不会叫“按钮权限”，而是叫：

```text
操作权限
功能权限
动作权限
接口权限
APP功能点权限
```

一句话理解：

> **APP 菜单权限控制“能不能看到入口”；APP 按钮权限控制“进来之后能不能操作”。**

---

# 1. APP 菜单权限和按钮权限有什么区别？

比如 B 端安装商 APP 有一个页面：

```text
设备详情页
```

用户能不能进入“设备详情页”，这是 **APP 菜单 / 页面权限**。

进入设备详情后，页面里可能有这些按钮：

```text
设备更名
重新配网
解绑设备
远程升级
下发控制
查看实时监控
导出数据
交付用户
```

用户能不能点这些按钮，就是 **APP 按钮权限 / 操作权限**。

所以权限可以分成两层：

| 权限类型     | 控制什么         | 例子                  |
| -------- | ------------ | ------------------- |
| APP 菜单权限 | 能不能看到入口 / 页面 | 设备管理、新增电站、任务中心      |
| APP 按钮权限 | 能不能执行操作      | 删除设备、解绑设备、远程升级、下发指令 |

---

# 2. 为什么 APP 也需要按钮权限？

因为 APP 不是只有 C 端用户在用，B 端安装商 APP、运维 APP、售后 APP 里有很多高风险操作。

比如 HaierEnergy B 端 APP：

```text
新增电站
扫码添加设备
BLE 配网
交付 C 端用户
重新配网
解绑设备
远程升级 OTA
下发控制指令
修改电站信息
删除设备
```

这些操作不是所有人都应该有。

举例：

## 安装工

可以：

```text
新增电站
扫码添加设备
BLE 配网
交付用户
查看自己负责的电站
```

不应该可以：

```text
删除设备
远程升级
解绑其他人的设备
查看全部安装商数据
```

## 安装商管理员

可以：

```text
查看本安装商下所有电站
分配安装工
交付用户
查看设备状态
```

部分高风险操作可能需要谨慎开放：

```text
远程升级
解绑设备
删除设备
批量操作
```

## 平台运维

可以：

```text
远程升级
下发诊断指令
查看告警
查看设备日志
```

但也不一定能：

```text
修改安装商组织
删除用户
修改财务数据
```

所以 APP 一样需要按钮权限。

---

# 3. C 端 APP 也需要按钮权限吗？

要分情况。

## C 端用户 APP 通常不叫“按钮权限”

C 端更常见的是：

```text
功能开关
会员权益
设备能力
用户角色
家庭共享权限
```

比如家庭能源 C 端 APP：

```text
查看电站数据
查看设备详情
修改设备名称
分享家庭成员
远程控制设备
查看收益报表
```

如果是普通家庭成员，可能只能查看，不能控制。

例如：

| 用户身份  | 可查看数据  | 可控制设备 | 可邀请成员 | 可解绑设备 |
| ----- | ------ | ----- | ----- | ----- |
| 家庭拥有者 | 可以     | 可以    | 可以    | 可以    |
| 家庭成员  | 可以     | 可配置   | 不可以   | 不可以   |
| 访客    | 可以部分查看 | 不可以   | 不可以   | 不可以   |

这本质上也是按钮权限，只是在 C 端产品里通常用“成员权限”“家庭权限”“功能权限”表达。

---

# 4. APP 按钮权限怎么实现？

和 Web 后台类似，本质还是：

```text
用户 → 角色 → 权限点
```

只不过权限点属于 APP 端。

比如后端返回 APP 权限：

```json
{
  "appMenus": [
    "station:create",
    "device:list",
    "task:center"
  ],
  "appPermissions": [
    "station:add",
    "device:scan",
    "device:config_network",
    "device:rename",
    "station:handover"
  ]
}
```

APP 根据权限决定按钮显示。

例如：

```text
有 device:rename → 显示“设备更名”
有 device:unbind → 显示“解绑设备”
有 device:ota → 显示“远程升级”
没有 device:delete → 不显示“删除设备”
```

但是重点是：

> **APP 隐藏按钮只是前端控制，后端接口也必须校验权限。**

否则用户可以绕过 APP，直接调用接口。

---

# 5. APP 权限应该怎么设计？

建议你把 APP 权限分成三类。

---

## 第一类：APP 菜单 / 页面权限

控制能不能看到功能入口。

例如：

```text
B端APP-首页
B端APP-电站管理
B端APP-新增电站
B端APP-设备管理
B端APP-任务中心
B端APP-我的
```

---

## 第二类：APP 操作 / 按钮权限

控制页面内能不能操作。

例如：

```text
电站-新增
电站-编辑
电站-删除
电站-交付用户
设备-扫码添加
设备-BLE配网
设备-重新配网
设备-解绑
设备-更名
设备-远程升级
设备-下发指令
设备-查看实时数据
设备-查看历史数据
```

---

## 第三类：APP 数据权限

控制能看哪些数据。

例如：

```text
全部电站
本安装商电站
本人创建电站
分配给我的电站
本部门及下级人员电站
```

这里特别重要：

> **APP 端权限不能只控制按钮，还要控制数据范围。**

比如安装工有“设备管理”入口，但只能看自己负责的设备，不应该看到全公司设备。

---

# 6. 哪些 APP 按钮必须做权限？

不是所有按钮都需要权限控制。

## 必须做权限的按钮

只要涉及以下类型，建议做权限：

```text
新增
编辑
删除
解绑
交付
远程升级
下发控制
导出
审核
分配
重置
禁用
启用
查看敏感信息
```

在 HaierEnergy 里尤其包括：

```text
新增电站
扫码添加设备
BLE 配网
重新配网
设备解绑
设备更名
交付 C 端用户
远程 OTA 升级
下发控制指令
查看设备日志
导出报表
```

---

## 不一定需要按钮权限的按钮

普通交互类按钮一般不需要单独配置：

```text
返回
刷新
展开/收起
切换 Tab
搜索
清空筛选
查看帮助
联系客服
```

但“搜索”本身不需要权限，搜索出来的数据范围需要受数据权限控制。

---

# 7. 产品需求里怎么写？

你可以这样写：

```text
B端 APP 支持按角色控制功能入口和操作权限。

1. APP 菜单权限：
控制用户是否可见 APP 内的功能入口，包括新增电站、设备管理、任务中心、交付用户等。

2. APP 操作权限：
控制用户在页面内是否可执行具体操作，包括扫码添加设备、BLE 配网、设备更名、重新配网、解绑设备、远程升级、交付用户等。

3. 数据权限：
控制用户在 APP 内可查看和操作的数据范围。安装工仅可查看本人创建或分配给本人的电站和设备；安装商管理员可查看本安装商下所有电站和设备。

4. 权限校验：
APP 根据后端返回的权限点控制按钮展示；所有高风险操作接口必须由后端再次校验权限。无权限时，接口返回无权限提示。
```

# 8. 真实例子：安装工角色

假设角色是：

```text
安装工
```

## APP 菜单权限

允许：

```text
首页
新增电站
我的电站
设备管理
扫码配网
任务中心
我的
```

不允许：

```text
组织管理
角色管理
权限管理
报表中心
OTA批量升级
```

## APP 按钮权限

允许：

```text
扫码添加设备
BLE 配网
重新配网
编辑本人创建的电站
交付 C 端用户
查看设备详情
查看实时数据
```

不允许：

```text
删除设备
解绑设备
远程升级
下发高风险控制指令
查看全部安装商设备
导出全部数据
```

## 数据权限

```text
仅本人创建或被分配的电站、设备、任务
```

这就是完整 APP 权限，而不是只控制菜单。

# 9. 和开发沟通时你可以这样问

```text
APP 端目前是否支持按钮/操作权限？
APP 菜单权限和按钮权限是同一套权限表，还是分开管理？
B端 APP 权限点是否有 permission_code？
APP 登录后后端是否返回权限点列表？
APP 隐藏按钮后，后端接口是否仍然校验权限？
安装工、安装商管理员、平台运维在 APP 端分别有哪些操作权限？
数据权限在 APP 列表接口中是否生效？
权限修改后，APP 是否需要重新登录才生效？
```

最关键要问一句：

```text
如果用户没有“设备解绑”权限，但直接调用解绑接口，后端会不会拦截？
```

如果答案是不会，那说明权限只是前端显示控制，不安全。

---

# 10. 最简单总结

**APP 有按钮权限，而且 B 端 APP 很需要。**

完整权限应该是：

```text
APP 菜单权限：能不能看到入口
APP 页面权限：能不能进入页面
APP 按钮权限：能不能执行操作
APP 接口权限：后端是否允许操作
APP 数据权限：能操作哪些数据
```

对 HaierEnergy 来说，像这些都应该纳入 APP 操作权限：

```text
新增电站
扫码添加设备
BLE 配网
重新配网
设备更名
解绑设备
交付用户
远程升级
下发控制
查看设备日志
```

一句话：

> **APP 不只是“显示几个入口”，只要有高风险操作，就应该有按钮/操作权限，并且必须由后端接口二次校验。**
