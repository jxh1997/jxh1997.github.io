问题：如果设备是4G或者有线联网，怎么完成设备在云平台注册？


4G 或有线联网的设备，**不需要用户给它下发 Wi-Fi 名称和密码**，但仍然需要完成：

```text
设备联网
↓
设备连接正确云平台
↓
设备向云平台证明身份
↓
云平台注册 / 激活设备
↓
业务系统绑定到用户 / 家庭 / 电站
```

所以核心区别是：

> **Wi-Fi / BLE 配网重点是“把网络信息给设备”；
> 4G / 有线联网重点是“设备自己有网后，如何自动连云、注册、绑定”。**

---

# 1. 4G / 有线设备还需要“配网”吗？

严格说，不一定需要传统意义上的 Wi-Fi 配网。

## Wi-Fi 设备

设备不知道现场 Wi-Fi，所以需要 APP 下发：

```text
Wi-Fi 名称
Wi-Fi 密码
云平台地址
设备绑定信息
```

## 4G 设备

设备插了 SIM 卡，通电后自己通过移动网络上网。

通常不需要用户输入 Wi-Fi。

## 有线设备

设备插上网线后，通过路由器自动获取网络。

通常也不需要用户输入 Wi-Fi。

所以 4G / 有线设备更多是：

```text
通电 / 插网线
↓
自动联网
↓
自动连接云平台
↓
自动注册
↓
APP / 后台完成绑定
```

产品上可以叫：

```text
设备激活
设备上线
设备注册
设备绑定
```

不一定叫“配网”。

---

# 2. 4G 设备如何完成云端注册？

以一台 4G 微储设备为例。

设备出厂时通常已经内置：

```text
设备 SN
设备型号
设备密钥 / 证书
云平台地址
通信协议
SIM 卡信息
APN 配置
```

完整流程是：

```text
设备通电
↓
4G 模组启动
↓
SIM 卡入网
↓
设备获得移动网络连接
↓
设备连接 HaierEnergy 云平台
↓
设备携带 SN / 证书 / 签名进行认证
↓
云平台校验设备合法性
↓
云平台完成注册 / 激活
↓
设备状态变为在线
↓
后端根据 SN 绑定到电站 / 用户
```

这里最重要的是：

> **4G 设备不需要 APP 下发 Wi-Fi，但设备必须提前知道要连接哪个云平台，并具备合法身份凭证。**

---

# 3. 有线设备如何完成云端注册？

有线设备通常是插网线后自动联网。

完整流程是：

```text
设备通电
↓
插入网线
↓
通过 DHCP 从路由器获取 IP
↓
检测是否能访问外网
↓
连接 HaierEnergy 云平台
↓
发送设备身份信息
↓
云平台认证
↓
设备注册 / 激活
↓
设备上线
↓
绑定到用户 / 电站
```

有线联网里常见关键点是：

```text
是否自动获取 IP
是否需要固定 IP
现场网络是否允许访问外网
防火墙是否拦截
DNS 是否正常
网线是否连接
路由器是否可用
```

所以有线设备的问题不在 Wi-Fi 密码，而在：

```text
网络是否通
能不能访问云
设备身份是否合法
业务是否已绑定
```

---

# 4. 4G / 有线设备的云平台注册方式

一般有三种模式。

---

## 模式一：出厂预注册

这是能源设备、工业设备里最常见、最安全的方式。

出厂前，厂商把设备资料导入 HaierEnergy 设备库：

```text
设备 SN
设备型号
设备密钥
证书
批次
厂商
销售区域
通信协议版本
```

设备第一次联网后，云平台校验这些信息。

流程：

```text
设备出厂前已在云端设备库
↓
设备联网
↓
设备携带 SN 和密钥连接云
↓
云平台校验通过
↓
设备激活
↓
设备上线
```

产品上可以理解为：

> 设备还没安装前，平台已经“认识”它；设备第一次上线时，只是来确认身份并激活。

适合：

```text
微逆
储能
PCS
BMS
网关
电表
充电桩
工业设备
```

---

## 模式二：首次上线动态注册

设备第一次连云时，云平台根据设备上报信息自动创建设备档案。

流程：

```text
设备联网
↓
上报 SN / 型号 / 签名
↓
云平台判断是否允许接入
↓
自动创建设备档案
↓
设备上线
```

这种方式更灵活，但安全要求高。

如果校验不严格，可能出现：

```text
非法设备接入
假 SN 冒充
测试设备混入生产环境
设备型号无法准确识别
```

所以能源类设备一般不建议完全无门槛动态注册。

---

## 模式三：APP / 后台预绑定后等待设备上线

这是安装工场景很常见的模式。

安装工先在 B 端 APP 创建电站，然后扫码录入设备 SN。

设备可能此时还没通电、没插网线、没插 SIM 卡。

流程：

```text
安装工创建电站
↓
扫码录入设备 SN
↓
系统校验 SN 合法
↓
设备状态 = 待上线 / 待激活
↓
设备现场通电联网
↓
设备自动连接云平台
↓
云平台注册成功
↓
系统自动把设备绑定到之前的电站
↓
设备状态 = 在线
```

这种模式非常适合 4G / 有线设备。

因为用户不需要在 APP 里一步步配 Wi-Fi，只需要：

```text
扫码添加设备
等待设备上线
```

---

# 5. APP 在 4G / 有线设备里做什么？

4G / 有线设备虽然不需要下发 Wi-Fi，但 APP 仍然很重要。

APP 主要负责：

```text
扫码识别设备 SN
校验设备是否支持接入
创建设备和电站的绑定关系
展示设备上线状态
提示安装工检查网络
展示注册失败原因
```

也就是说，APP 不一定参与“联网”，但参与“业务绑定”。

典型流程：

```text
安装工扫码设备
↓
APP 获取 SN
↓
后端校验设备合法性
↓
安装工选择电站
↓
后端创建预绑定关系
↓
页面提示：等待设备上线
↓
设备通电后自动注册云端
↓
APP 显示：设备已上线
```

---

# 6. 4G / 有线设备配网成功标准是什么？

建议不要叫“配网成功”，可以叫：

```text
设备上线成功
设备激活成功
设备添加成功
```

完整成功标准建议是：

```text
1. 设备已连接网络；
2. 设备已连接 HaierEnergy 云平台；
3. 云平台完成设备身份认证；
4. 设备注册 / 激活成功；
5. 设备与目标电站绑定成功；
6. APP 可拉取到设备基础信息和在线状态。
```

如果只是设备通电，不算成功。

如果只是插了网线，也不算成功。

如果只是 4G 模组入网，也不算成功。

真正成功应该是：

> **云平台确认设备上线，并完成业务绑定。**

# 7. 4G 设备常见异常场景

4G 设备失败一般不是 Wi-Fi 密码问题，而是这些问题。

| 异常        | 原因         | 产品提示                |
| --------- | ---------- | ------------------- |
| SIM 未激活   | 卡未开通或套餐异常  | 请确认设备 SIM 卡已激活      |
| 4G 信号弱    | 现场信号差      | 请将设备安装在信号较好的位置      |
| APN 错误    | 运营商接入点配置不对 | 设备移动网络接入失败，请联系安装商   |
| 欠费 / 流量耗尽 | SIM 无法联网   | SIM 卡网络不可用，请检查流量或套餐 |
| 云平台不可达    | 网络可用但访问不了云 | 设备已联网，但无法连接云平台      |
| 证书错误      | 设备认证失败     | 设备认证失败，请联系售后        |
| SN 未入库    | 云端不认识设备    | 设备未录入系统，请确认 SN      |

产品上不要统一提示“设备上线失败”，要尽量按阶段提示。

---

# 8. 有线设备常见异常场景

有线设备常见问题是网络环境和现场布线。

| 异常         | 原因               | 产品提示            |
| ---------- | ---------------- | --------------- |
| 未插网线       | 物理连接未建立          | 请检查网线是否插好       |
| 路由器未分配 IP  | DHCP 异常          | 请确认路由器 DHCP 已开启 |
| 固定 IP 配置错误 | IP / 网关 / DNS 错误 | 请检查网络参数         |
| 无外网        | 局域网通，但不能访问互联网    | 当前网络无法访问互联网     |
| 防火墙拦截      | 企业网络限制端口         | 请开放设备访问云平台所需端口  |
| DNS 失败     | 域名解析异常           | 请检查网络 DNS 配置    |
| 证书认证失败     | 设备身份不合法          | 设备认证失败，请联系售后    |

有线设备尤其要注意：有些工商业现场网络可能有防火墙、白名单、代理限制。

---

# 9. 4G / 有线设备如何知道连接哪个云？

这和 Wi-Fi 设备一样，核心仍然是云平台连接参数。

常见方式：

```text
1. 固件出厂写死 HaierEnergy 云地址
2. 设备内置引导服务器地址，由引导服务器分配目标云
3. 根据 SIM / 区域 / SN 判断连接哪个区域云
4. 通过本地配置工具或运维平台修改云地址
5. OTA 修改云平台地址
```

4G / 有线设备没有 Wi-Fi 配网过程，所以如果要让它连接正确云平台，通常更依赖：

```text
出厂固件配置
设备证书
区域规则
引导服务器
后台设备库
```

产品经理要问清楚：

```text
设备云地址是出厂写死，还是可远程配置？
不同国家设备是否连接不同区域云？
设备如果发往错误区域，是否能重新分配云？
测试设备和生产设备如何隔离？
```

# 10. 4G / 有线设备与 Wi-Fi 配网最大的区别

| 对比项            | Wi-Fi / BLE 配网设备    | 4G / 有线设备          |
| -------------- | ------------------- | ------------------ |
| 是否需要用户输入 Wi-Fi | 需要                  | 通常不需要              |
| APP 是否下发网络信息   | 通常需要                | 通常不需要              |
| 设备如何联网         | 连接现场 Wi-Fi          | 4G 入网 / 网线联网       |
| APP 主要作用       | 下发网络 + 绑定           | 扫码添加 + 绑定 + 等待上线   |
| 常见失败原因         | Wi-Fi 密码错、2.4G、蓝牙失败 | SIM、信号、网线、DHCP、防火墙 |
| 成功标准           | 设备连云并绑定             | 设备连云并绑定            |
| 云端注册是否需要       | 需要                  | 同样需要               |

注意最后一项：

> **无论 Wi-Fi、4G 还是有线，只要设备要接入云平台，都需要云端注册。**

# 11. 产品需求里怎么写？

你可以这样写一段 PRD。

```text
设备联网与云端注册规则：

1. 4G / 有线设备不需要用户输入 Wi-Fi 信息。

2. 安装工通过扫描设备二维码录入设备 SN，系统校验设备 SN 是否存在、型号是否支持、是否已被绑定。

3. 校验通过后，系统允许将设备预绑定到当前电站，设备状态显示为“待上线”。

4. 设备通电并完成 4G / 有线联网后，自动连接 HaierEnergy 云平台。

5. 云平台根据设备 SN、设备密钥 / 证书、产品型号完成身份认证。

6. 认证通过后，设备状态变为“已注册 / 已上线”。

7. 若设备已预绑定电站，则系统自动完成设备与电站的激活关系，设备状态变为“在线”。

8. 若设备未预绑定，则设备进入“待绑定设备池”，需由用户或安装工绑定后方可展示在电站中。

9. 设备上线等待超时时间为 120 秒。超时后提示用户检查 SIM 卡、信号、网线、路由器和现场网络。
```

# 12. 页面流程可以怎么设计？

## 安装工端添加 4G / 有线设备

```text
进入电站建站流程
↓
点击添加设备
↓
选择设备联网方式：4G / 有线 / Wi-Fi
↓
扫码识别设备 SN
↓
后端校验设备合法性
↓
绑定到当前电站
↓
页面显示：等待设备上线

设备上线成功
↓
页面显示：添加成功
```

如果设备暂时没有上线：

```text
设备已添加，等待上线
请确认设备已通电并完成联网
```

# 13. 状态设计建议

4G / 有线设备可以设计这些状态：

```text
未添加
已预绑定
待上线
注册中
已上线
在线
离线
注册失败
绑定失败
禁用
```

更适合用户理解的展示文案：

| 技术状态            | 用户展示      |
| --------------- | --------- |
| pre_bound       | 已添加，待设备上线 |
| registering     | 设备注册中     |
| registered      | 设备已注册     |
| online          | 在线        |
| offline         | 离线        |
| register_failed | 注册失败      |
| bind_failed     | 绑定失败      |

# 14. 你应该怎么问开发？

你可以这样问：

```text
4G / 有线设备出厂时是否已内置 HaierEnergy 云地址？
设备 SN 和密钥是否已提前导入云端设备库？
设备首次联网后是预注册激活，还是动态注册？
设备使用什么协议连接云平台？MQTT、HTTPS 还是 TCP？
云平台如何校验设备身份？SN、deviceSecret、证书还是签名？
设备上线后是否有 deviceOnline 事件或回调？
设备注册成功和设备绑定成功是否是两个独立动作？
设备未提前绑定时，上线后进入哪里？待绑定池吗？
安装工是否允许离线预绑定设备？
4G 设备 SIM 卡/APN/信号异常时，设备是否能返回错误码？
有线设备 DHCP、DNS、防火墙异常时，如何定位？
```

# 15. 最简单总结

4G / 有线设备完成云端注册，不靠用户输入 Wi-Fi，而靠设备自动联网后主动连接云平台。

核心流程是：

```text
设备通电
↓
4G / 有线联网
↓
连接 HaierEnergy 云
↓
携带 SN / 密钥 / 证书认证
↓
云平台注册 / 激活设备
↓
设备上线
↓
绑定到用户 / 电站
```

对产品经理来说，最关键的不是“怎么联网”，而是定义清楚：

```text
设备何时算上线？
何时算注册成功？
是否允许离线预绑定？
设备上线后如何自动绑定到电站？
注册失败如何提示？
4G / 有线异常如何排查？
```

一句话：

> **4G / 有线设备省掉了 Wi-Fi 配网步骤，但没有省掉云端注册、身份认证和业务绑定。**
